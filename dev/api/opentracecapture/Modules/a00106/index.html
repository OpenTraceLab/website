    function fd_source_new¶

    static GSource * fd_source_new(
    struct otc_session * session,
    void * key,
    gintptr fd,
    int events,
    int timeout_ms
)

    Create an event source for I/O on a file descriptor.

    Parameters:

    • session The session the event source belongs to.
    • key The key used to identify this source.
    • fd The file descriptor or HANDLE.
    • events Events.
    • timeout_ms The timeout interval in ms, or -1 to wait indefinitely.

    Return: A new event source object, or NULL on failure.

    In order to maintain API compatibility, this event source also doubles as a timer event source.

    function set_main_context¶

    static int set_main_context(
    struct otc_session * session
)

    Set up the main context the session will be executing in.

    Must be called just before the session starts, by the thread which will execute the session main loop. Once acquired, the main context pointer is immutable for the duration of the session run.

    function unset_main_context¶

    static int unset_main_context(
    struct otc_session * session
)

    Unset the main context used for the current session run.

    Must be called right after stopping the session. Note that if the session is stopped asynchronously, the main loop may still be running after the main context has been unset. This is OK as long as no new event sources are created – the main loop holds its own reference to the main context.

    function otc_session_start¶

    OTC_APIint otc_session_start(
    struct otc_session * session
)

    Start a session.

    Parameters:

    • session The session to use. Must not be NULL.

    Returns:

    • OTC_OK Success.
    • OTC_ERR_ARG Invalid session passed.
    • OTC_ERR Other error.

    Since: 0.4.0

    When this function returns with a status code indicating success, the session is running. Use otc_session_stopped_callback_set() to receive notification upon completion, or call otc_session_run() to block until the session stops.

    Session events will be processed in the context of the current thread. If a thread-default GLib main context has been set, and is not owned by any other thread, it will be used. Otherwise, libopentracecapture will create its own main context for the current thread.

    function otc_session_run¶

    OTC_APIint otc_session_run(
    struct otc_session * session
)

    Block until the running session stops.

    Parameters:

    • session The session to use. Must not be NULL.

    Returns:

    • OTC_OK Success.
    • OTC_ERR_ARG Invalid session passed.
    • OTC_ERR Other error.

    Since: 0.4.0

    This is a convenience function which creates a GLib main loop and runs it to process session events until the session stops.

    Instead of using this function, applications may run their own GLib main loop, and use otc_session_stopped_callback_set() to receive notification when the session finished running.

    function otc_session_stop¶

    OTC_APIint otc_session_stop(
    struct otc_session * session
)

    Stop a session.

    Parameters:

    • session The session to use. Must not be NULL.

    Returns:

    • OTC_OK Success.
    • OTC_ERR_ARG Invalid session passed.

    Since: 0.4.0

    This requests the drivers of each device participating in the session to abort the acquisition as soon as possible. Even after this function returns, event processing still continues until all devices have actually stopped.

    Use otc_session_stopped_callback_set() to receive notification when the event processing finished.

    This function is reentrant. That is, it may be called from a different thread than the one executing the session, as long as it can be ensured that the session object is valid.

    If the session is not running, otc_session_stop() silently does nothing.

    function otc_session_is_running¶

    OTC_APIint otc_session_is_running(
    struct otc_session * session
)

    Return whether the session is currently running.

    Parameters:

    • session The session to use. Must not be NULL.

    Returns:

    • TRUE Session is running.
    • FALSE Session is not running.
    • OTC_ERR_ARG Invalid session passed.

    Since: 0.4.0

    Note that this function should be called from the same thread the session was started in.

    function otc_session_stopped_callback_set¶

    OTC_APIint otc_session_stopped_callback_set(
    struct otc_session * session,
    otc_session_stopped_callback cb,
    void * cb_data
)

    Set the callback to be invoked after a session stopped running.

    Parameters:

    • session The session to use. Must not be NULL.
    • cb The callback to invoke on session stop. May be NULL to unset.
    • cb_data User data pointer to be passed to the callback.

    Returns:

    • OTC_OK Success.
    • OTC_ERR_ARG Invalid session passed.

    Since: 0.4.0

    Install a callback to receive notification when a session run stopped. This can be used to integrate session execution with an existing main loop, without having to block in otc_session_run().

    Note that the callback will be invoked in the context of the thread that calls otc_session_start().
